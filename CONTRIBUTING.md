# Contributing to Owlet CMS 🦉

Thank you for your interest in contributing to Owlet!

Owlet is an open-source project focused on helping libraries, archives, and community knowledge spaces build better websites and digital collections. Contributions of all kinds are welcome.

## Ways to Contribute

You don't have to write code to help the project. Useful contributions include:

* reporting bugs
* suggesting features
* improving documentation
* sharing library workflows and real-world needs
* contributing plugins
* improving accessibility
* UI/UX design ideas

Libraries and community institutions have many unique needs, and your experience can help shape the platform.

---

## Development Setup

Prerequisites:

* Node.js
* Docker
* Git

Clone the repository:

```
git clone https://github.com/yourusername/owlet-cms.git
cd owlet-cms
```

Start the backend:

```
cd backend
npm install
npm run start
```

Start the frontend:

```
cd frontend
npm install
npm run dev
```

More development setup instructions will be added as the project evolves.

---

## Contribution Guidelines

When contributing code:

1. Keep pull requests focused and small.
2. Write clear commit messages.
3. Add documentation when appropriate.
4. Discuss large changes in issues before implementing them.

Example commit messages:

```
Add initial events API
Create media upload service
Add form submission email receipts
```

---

## Code Philosophy

Owlet prioritizes:

* readability
* maintainability
* accessibility
* long-term stability

We prefer **simple, understandable solutions** over clever or overly complex implementations.

---

## Community Expectations

Please be respectful and constructive.

Owlet aims to be a welcoming project for:

* developers
* librarians
* archivists
* designers
* community technologists

Different perspectives help the platform grow in useful ways.
